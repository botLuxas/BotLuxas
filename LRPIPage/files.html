What these files do

<nominal>2</nominal>: This is the nominal (or default) quantity of the item in the world. In this case, the game will try to have 2 of this item at a time in the world.

<lifetime>14400</lifetime>: The lifetime of this item is set to 14,400 seconds (or 4 hours). This means after 4 hours, the item will despawn or be removed from the game world unless interacted with.

<restock>1800</restock>: This refers to how often the item is restocked in the game world. The value is set to 1,800 seconds (or 30 minutes), meaning the item can respawn in the game world every 30 minutes.

<min>1</min>: The minimum number of items that should exist in the world. In this case, at least 1 of this item should always be present.

<quantmin>-1</quantmin> & <quantmax>-1</quantmax>: These define the minimum and maximum quantities for the item, but since both are set to -1, it means there are no restrictions on the amount of the item that can exist. It can be any quantity.

<category name=""/>: This would define a category for the item, but it's left empty in this case, meaning itâ€™s not categorized.

<tag name="floor"/>: The item is tagged as being associated with the floor, so it's probably something that can be placed or found on the floor in buildings or structures.

<usage name="Coast"/>, <usage name="Farm"/>, <usage name="Industrial"/>, etc.: These define the locations or environments where the item is used or found. It will be present in coastal areas, farms, industrial zones, offices, schools, towns, and villages.

<value name="Tier1"/>, <value name="Tier2"/>, <value name="Tier3"/>: These define the tiers or levels the item belongs to. The item is available in all three tiers, meaning it could be found in low, medium, or high-tier loot areas or serve as an item for different gameplay stages.
